Unit testing - Протестувати кожну частину програми окремо. Наприклад відкриваємо вікно 
для створеня нового повідомлення:
Бачимо перед собою вікно для створення нового повідомлення зверху яког розміщено панель керування
і панель інструментів, під цими панелями розміщено поля для введення адреси одержувача та теми
повідомлення, ще нижче можна вводитити сам тест повідомлення. програма одночасно дозволяє вводити до 
трьох одержувачів. Існує можливість прикріплювати вкладення, встановлювати захист на повідомлення та 
перевіряти на орфографію. Потрібно також протестувати всі ці можливості по окремо Unit тестуванням.

Integration testing - Протестувати вцілому функціонали які дозволяє програма. Наприклад:
перевірити повністю процес створення і відправки нового повідомлення, перевірки повідомлень, фільтрації,
перегляду подій та створення нових подій, пошук потекстах повідомлень, та інше.

System testing - Пройтись по всіх випадках які б могли зустрітись користувачу, починаючи з логування
до написання повідомлень створення чату тощо, на кожен окремий випадок потрібно писати окремий тест кейс.
Другим аспектом являється перевірка зручності використання програми. 
Дана програма досить зручна у використанні за вийнятком того що спочатку ідуть повідомлення посортовані по даті від старих со новіших.

Smoke Testing - Залогінитись, попробувати почитати повідомлення, переглянути вкладення написати і відправити повідомлення.

Functional testing - Первірити на на правельність бізнес логіку програми у різних випадках. наприклад: на введенні неправильних символів
у полях логінації. Перевірити як буде поводити себе програма при введенні різного типу даних у поля пошуку та фільтрування, 
та як воно буде відбуватись, перевірити фільтрацію непрочитаних та важливих повідомлень.

GUI testing - Графічний інтерфейс додатку додатку дружелюбний, легко можна знайти те що потібно, Шрифти рівні приємні для перегляду, 
непрочтані повідомлення виділяються жирним шрифтом, що є зручнішим ніж використання певних позначень.

Globalization, Internationalization testing - При встановленні програми одразу підтягнуло мої дані геолокації і встаноло її українською мовою, хоча був вибір
встановити і іншими мовами, при тестуванні глобалізації багів не було виявлено.
Виявлено баг коли робити загальний пошук по повідомленнях то повідомлення у яких є тестова інформаці явідображаються коректно,
а повідомлення з використанням Html або інших засобів відображаються не дружньо для користувача(у вигляді розмітки).

 
Localization testing - при тестуванні всі повідомлення, всі розділи, назви, підказки та інше перекладено правельно. 
Повідомлення отримані або відравлення написані мовою оригіналу, формат дати відповідає формату дати у даному регіоні.

Usability testing - У процесі тестування програма досить швидко відгукувалась, кожне віткрите пвідомлення відкривалось одразу у новій,
що досить зручно для перегляду одразу декількох повідомлень.
Не зручним виявилось відкриття папки на google диску, яке відкривалось у браузері по замовчуванню.
Перегляд вкладень зручний, вкладення текстових файлів автоматично відкриваються нижче тексту повідомлення.

Acceptance testing - Сереньо статестичний користувач зможе легко розібратись з програмою і користуватись нею. 
Програма є зручною і простою для користування.
